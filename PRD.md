## 一、背景与目标

### 1.1 背景

日常消费场景中，用户通过支付宝、微信等多个支付平台进行消费，存在以下痛点：
- **数据分散**：消费记录分散在多个平台，无法统一查看
- **缺乏标注**：无法为每笔消费添加自定义标签，时间久后遗忘消费用途
- **统计困难**：无法获得整体收支概览和趋势分析

### 1.2 产品目标

构建一款个人记账助手，实现：
- 统一管理多来源的收支记录
- 支持灵活的分类标签体系
- 提供直观的统计报表和趋势分析
- 支持移动端和 Web 端多平台使用

### 1.3 成功指标

| 指标 | 目标值 |
|------|--------|
| 记账效率 | 单笔记录录入时间 < 30秒 |
| 数据完整性 | 每月记账覆盖率 > 90% |
| 使用频率 | 每周活跃使用 ≥ 3次 |

### 2.2 交易类型与账户变动规则

| 交易类型 | 账户变动 | 收支影响 | 统计归类 |
|---------|---------|---------|---------|
| **支出** | 账户余额 - 金额 | 总支出 + 金额 | 支出统计 |
| **收入** | 账户余额 + 金额 | 总收入 + 金额 | 收入统计 |
| **转账** | 转出账户 - 金额，转入账户 + 金额 | 无影响 | 不计入收支 |
| **退款** | 账户余额 + 金额 | 总支出 - 金额 | 支出减少 |
| **借出** | 账户余额 - 金额 | 无影响 | 借出统计 |
| **借入** | 账户余额 + 金额 | 无影响 | 借入统计 |

### 2.3 异常处理流程

| 异常场景 | 处理方式 |
|---------|---------|
| 账单文件格式错误 | 提示"文件格式不支持，请导出正确的CSV格式" |
| 账单解析失败 | 提示具体失败行号，支持跳过错误行继续导入 |
| 余额不足（转账/支出） | 允许余额为负，显示警告提示 |
| 分类匹配失败 | 引导用户手动选择分类，并记录匹配规则供后续学习 |
| 数据导入冲突 | 检测重复记录（相同日期+金额+账户），提示用户选择覆盖或跳过 |

## 三、功能详细说明

### 3.1 账户管理

#### 3.1.1 功能概述

管理用户的资金账户，支持预设账户和自定义账户，追踪各账户余额。

#### 3.1.2 交互规范

**预设账户列表：**

| 账户名称 | 初始图标 | 默认币种 | 可删除 |
|---------|---------|---------|--------|
| 支付宝 | 蓝色图标 | CNY | 否 |
| 微信支付 | 绿色图标 | CNY | 否 |
| 银行卡 | 金色图标 | CNY | 否 |
| 现金 | 灰色图标 | CNY | 否 |
| 信用卡 | 紫色图标 | CNY | 否 |

**账户操作：**

| 操作 | 触发条件 | 交互行为 |
|------|---------|---------|
| 新增账户 | 点击"添加账户"按钮 | 弹出表单：账户名称、图标选择、初始余额、币种（默认CNY） |
| 编辑账户 | 长按/点击编辑图标 | 可修改名称、图标，余额不可直接修改（通过交易记录变更） |
| 删除账户 | 点击删除按钮 | 二次确认：该账户下有N条交易记录，是否确认删除？删除后记录保留，账户显示为"已删除" |
| 查看账户详情 | 点击账户卡片 | 展示账户余额、近期交易流水 |

#### 3.1.3 业务规则

- 账户名称不可重复
- 预设账户不可删除，但可隐藏（余额为0且无交易记录时）
- 账户余额 = 初始余额 + 所有入账 - 所有出账
- 币种默认为 CNY，V1.0 版本仅支持单币种

### 3.2 分类管理

#### 3.2.1 功能概述

管理收支分类，支持预设分类和用户自定义分类，用于交易记录的分类标注。

#### 3.2.2 预设分类体系

**支出分类：**

| 一级分类 | 二级分类（示例） | 图标 |
|---------|----------------|------|
| 餐饮 | 早餐、午餐、晚餐、夜宵、饮料 | 🍚 |
| 交通 | 公交、地铁、打车、共享单车、加油 | 🚗 |
| 购物 | 日用品、服饰、数码、家电 | 🛒 |
| 娱乐 | 电影、游戏、旅游、运动 | 🎮 |
| 居住 | 房租、水电、物业、装修 | 🏠 |
| 医疗 | 药品、门诊、住院 | 💊 |
| 教育 | 书籍、课程、培训 | 📚 |
| 社交 | 红包、礼物、请客 | 🎁 |
| 其他 | - | 📝 |

**收入分类：**

| 一级分类 | 二级分类（示例） | 图标 |
|---------|----------------|------|
| 工资 | 基本工资、奖金、津贴 | 💰 |
| 兼职 | 兼职收入、稿费 | 💵 |
| 投资 | 股票、基金、理财 | 📈 |
| 其他 | - | 📝 |

#### 3.2.3 交互规范

| 操作 | 触发条件 | 交互行为 |
|------|---------|---------|
| 新增分类 | 点击"添加分类" | 选择一级/二级分类，输入名称，选择图标 |
| 编辑分类 | 长按/点击编辑 | 可修改名称、图标 |
| 删除分类 | 点击删除 | 二次确认；已有交易记录的分类不可删除，提示"该分类下有N条记录，无法删除" |

#### 3.2.4 业务规则

- 分类名称在同一层级不可重复
- 预设分类不可删除，但可编辑名称和图标
- 交易记录必须关联一个二级分类

### 3.3 交易记录

#### 3.3.1 手动录入

**交互规范：**

| 字段 | 必填 | 默认值 | 输入方式 |
|------|------|--------|---------|
| 交易类型 | ✅ | 支出 | 底部 Tab 切换（支出/收入/转账/退款/借贷） |
| 金额 | ✅ | - | 数字键盘，支持小数点后两位 |
| 账户 | ✅ | 上次使用账户 | 下拉选择 |
| 分类 | ✅ | - | 弹窗选择（一级→二级） |
| 日期 | ✅ | 当前日期 | 日期选择器 |
| 时间 | ❌ | 当前时间 | 时间选择器 |
| 备注 | ❌ | - | 文本输入，最多100字 |

**转账类型特殊字段：**

| 字段 | 说明 |
|------|------|
| 转出账户 | 资金来源账户 |
| 转入账户 | 资金目标账户 |
| 金额 | 转账金额 |

**退款类型特殊字段：**

| 字段 | 说明 |
|------|------|
| 关联原支出 | 可选，关联历史支出记录 |
| 退款账户 | 退款到账账户 |

#### 3.3.2 账单导入

**支持的账单来源：**

| 平台 | 文件格式 | 导出路径参考 |
|------|---------|-------------|
| 支付宝 | CSV | 支付宝App → 我的 → 账单 → 导出 |
| 微信 | CSV | 微信App → 我 → 服务 → 钱包 → 账单 → 导出 |

**导入流程：**

```
选择文件 → 解析文件 → 字段映射 → 智能分类 → 预览确认 → 保存
```

**字段映射规则：**

| 账单字段 | 系统字段 | 映射说明 |
|---------|---------|---------|
| 交易时间 | 日期+时间 | 自动解析 |
| 交易金额 | 金额 | 支出为负，收入为正 |
| 交易对方 | 备注 | 追加到备注 |
| 商品说明 | 备注 | 用于智能分类匹配 |
| 收/支 | 交易类型 | 收入/支出/不计入（转账） |
| 支付方式 | 账户 | 匹配已有账户名称 |

**智能分类匹配规则：**

1. 精确匹配：商品说明包含分类关键词（如"麦当劳"→餐饮）
2. 模糊匹配：商品说明包含相关词汇（如"超市"→购物）
3. 学习匹配：用户历史手动选择记录（如"某某店铺"→餐饮）
4. 匹配失败：标记为"待分类"，需手动选择

#### 3.3.3 交易列表

**列表展示：**

| 元素 | 说明 |
|------|------|
| 日期分组 | 按日期分组，显示当日收支小计 |
| 交易项 | 图标 + 分类名称 + 备注 + 金额（支出红色，收入绿色） |
| 筛选 | 按日期范围、账户、分类、交易类型筛选 |
| 搜索 | 按备注内容搜索 |

**交易详情：**

| 操作 | 说明 |
|------|------|
| 编辑 | 修改交易信息 |
| 删除 | 二次确认后删除，恢复账户余额 |
| 复制 | 快速创建相似交易 |

### 3.4 数据同步

#### 3.4.1 数据导出

**导出格式：** JSON 文件（结构化数据，便于导入）

**导出内容：**
- 所有账户信息
- 所有分类信息
- 所有交易记录
- 应用设置

**导出路径：** 用户自选本地路径

#### 3.4.2 数据导入

**导入逻辑：**

| 场景 | 处理方式 |
|------|---------|
| 全新导入 | 直接导入所有数据 |
| 覆盖导入 | 清空现有数据，导入新数据 |
| 合并导入 | 检测重复记录，跳过或覆盖（用户选择） |

### 3.5 统计报表

#### 3.5.1 基础统计

**收支概览卡片：**

| 指标 | 计算方式 |
|------|---------|
| 本月支出 | 本月所有支出类型交易金额之和 |
| 本月收入 | 本月所有收入类型交易金额之和 |
| 本月结余 | 本月收入 - 本月支出 |
| 账户总资产 | 所有账户余额之和 |

**分类占比（饼图）：**

- 支出分类占比：各支出分类金额 / 总支出
- 收入分类占比：各收入分类金额 / 总收入
- 支持点击分类查看明细

#### 3.5.2 趋势分析

**收支趋势（折线图）：**

| 维度 | 说明 |
|------|------|
| 日趋势 | 近7日/30日每日收支折线 |
| 周趋势 | 近12周每周收支折线 |
| 月趋势 | 近12个月每月收支折线 |

**同比环比：**

| 指标 | 说明 |
|------|------|
| 环比 | 与上一周期对比（如本月 vs 上月） |
| 同比 | 与去年同期对比（如本月 vs 去年同月） |

## 五、技术方案建议

### 5.1 技术栈推荐

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| 移动端 | Flutter / React Native | 跨平台，一套代码支持 iOS 和 Android |
| Web端 | Vue 3 / React | 组件化开发，生态成熟 |
| 本地存储 | SQLite (移动端) / IndexedDB (Web) | 结构化数据存储，支持复杂查询 |
| 状态管理 | Provider / Redux | 数据状态同步 |

### 5.2 数据模型设计

**核心实体：**

```
Account（账户）
├── id: string
├── name: string
├── icon: string
├── currency: string
├── initialBalance: number
├── currentBalance: number
├── isPreset: boolean
└── createdAt: datetime

Category（分类）
├── id: string
├── name: string
├── icon: string
├── type: enum[expense, income]
├── parentId: string (nullable)
├── isPreset: boolean
└── createdAt: datetime

Transaction（交易记录）
├── id: string
├── type: enum[expense, income, transfer, refund, lend, borrow]
├── amount: number
├── accountId: string
├── toAccountId: string (nullable, for transfer)
├── categoryId: string
├── date: date
├── time: time
├── note: string
├── source: enum[manual, import]
└── createdAt: datetime
```
